# Supervisor Summary — LLM-Augmented NIDS Thesis

## February 2026

This thesis investigates whether Large Language Models can perform network intrusion detection through structured reasoning over NetFlow features, and what architecture maximises their effectiveness. The work began with a straightforward hypothesis: that connecting an LLM agent to external security tools via Anthropic's Model Context Protocol would improve detection performance by giving the model access to threat intelligence, geolocation data, and MITRE ATT&CK mappings. What we found was more interesting than what we expected.

We built a complete MCP-based NIDS: a Claude Sonnet agent connected to 13 security tools (AbuseIPDB, AlienVault OTX, IP geolocation, MITRE ATT&CK mapping, and a custom NetFlow behavioural analyser) via STDIO transport. The agent operates in a ReAct reasoning loop — observe flow features, decide which tools to call, interpret results, revise if needed, produce a verdict with confidence and natural-language explanation. We evaluated this on 58 stratified flows from the CICIDS2018 NetFlow v3 dataset, deliberately constructed as a four-tier test batch: flows with public IPs that should return threat intelligence, feature-rich volume attacks, temporal-context attacks requiring cross-flow correlation, and benign traffic.

Phase 1 produced a genuine surprise. The agent achieved 83.3% F1 on attack detection — but external threat intelligence returned zero meaningful data across all 142 queries. Every AbuseIPDB lookup, every OTX check, every geolocation call came back empty because CICIDS2018 uses RFC 1918 private addresses and expired 2018 AWS IPs. The agent's entire detection capability came from its pre-trained knowledge of what network attacks look like in flow features. This is not a failure of the architecture — the MCP integration works correctly — but a finding about where LLM capability actually resides. The model knows, from its training corpus, that 232× byte ratios and 599× packet counts relative to normal traffic indicate volumetric attacks. It does not need external tools to tell it this.

Phase 2 attempted to improve the baseline through three iterations: temporal memory (SQLite per-IP history), benign calibration (system prompt templates encouraging benefit-of-the-doubt), and a combined approach with confidence override. Temporal memory dramatically improved benign accuracy from 13.3% to 66.7% but introduced attack normalisation — the agent learned to treat repeated IP appearances as evidence of legitimacy. Benign calibration improved precision but reduced recall. The combined approach, Iteration 3, was catastrophic: 41.4% accuracy and 45.2% F1 as the confidence override suppressed 11 true positives while only correctly converting 6 false positives. This demonstrated a fundamental single-agent ceiling — no prompt configuration simultaneously optimises precision, recall, and benign accuracy because each modification creates conflicting decision pressures within the same context window.

These findings motivated a research pivot. The question shifted from "Can MCP tools improve LLM-based NIDS?" to "Can a multi-agent architecture of pure LLM reasoning, without external tools, match analyst-level detection?" We designed AMATAS (Autonomous Multi-Agent Threat Analysis System): six specialised agents — Protocol Semantics, Statistical Pattern, Behavioural Fingerprint, Temporal Correlation, Devil's Advocate, and an Orchestrator — each addressing a specific Phase 1–2 failure mode. The key architectural insight is decomposition: instead of one agent trying to balance competing objectives, each agent does one thing well, and disagreements are resolved through structured deliberation rather than brittle thresholds.

Phase 3 evaluated AMATAS across three configurations. Phase 3a (the same 58 flows) achieved 67.2% accuracy with 100% precision and 100% benign accuracy — zero false positives, at the cost of 55.8% recall. Phase 3b (150 flows) was the standout: 95.9% F1 with 100% recall and 92.1% precision, demonstrating that the temporal agent needs batch depth to function. Phase 3c (100 flows, 88% benign — realistic distribution) exposed the remaining weakness: 54% accuracy with 41 false positives, as specialist agents over-flag benign traffic when it dominates the batch.

Phase 3e investigated this directly. Increasing the Devil's Advocate orchestrator weight from 30% to 50% on the same batch corrected 6 false positives with zero harm to recall — a surgically precise improvement. But 34 of the remaining 35 false positives had unanimous 4/4 specialist agreement, meaning the problem is not orchestrator-level weighting but specialist-level priors. This is an honest and useful finding: it narrows the optimisation target from the whole system to specialist calibration for benign-heavy distributions, and it confirms that the DA weight is a meaningful architectural parameter rather than noise.

The open questions going forward are concrete. First, can specialist-level calibration — either distribution-aware prompting or a two-pass architecture where agents observe the batch before classifying — resolve the false positive problem on realistic distributions? Second, does AMATAS generalise beyond CICIDS2018? The dataset has documented flow construction errors and class imbalance; cross-validation on UNSW-NB15 is essential. Third, the scalability question remains: at $0.07 per flow, this is a proof of capability, not a production system. The architectural blueprint for cost reduction exists (hierarchical triage routing 90%+ of flows through a lightweight pre-filter), but it has not been built or tested. These are genuine research questions, not hand-waving — each has a clear experimental design and expected outcome.
